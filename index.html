CIDR & Subnetting FAQ

What is CIDR?

▼

CIDR (Classless Inter-Domain Routing) is a method for allocating IP addresses and routing IP packets. It replaces the older system based on classes A, B, and C. CIDR notation combines the network address with a suffix indicating the number of bits in the network prefix, e.g., 192.168.1.0/24.

Why is CIDR notation useful?

▼

CIDR notation offers a more flexible and efficient way to allocate IP addresses compared to the older classful addressing. It allows network administrators to allocate address blocks of exactly the right size, reducing address waste. It also simplifies routing tables by allowing route aggregation.

How is an IP address structured?

▼

An IPv4 address consists of 32 bits, usually represented as four 8-bit octets (bytes) in decimal form separated by dots (e.g., 192.168.1.1). Each octet can have a value from 0 to 255. The address is divided into network and host portions, with the CIDR prefix length determining how many bits are used for the network part.

What does the number after the slash mean?

▼

The number after the slash in CIDR notation (e.g., /24 in 192.168.1.0/24) represents the number of bits used for the network portion of the address. For example, /24 means the first 24 bits (first three octets) identify the network, while the remaining 8 bits are available for host addresses. This determines the size of the network - a smaller number after the slash means a larger network.
